Making your first Open Source Contribution

Basically I am using GitHub like Dropbox or Google Drive. I am ashamed to say that I have not done any contribution to the open-source projects, mainly because I am scared of screwing the project up (even though I know that the Git Workflow makes it impossible to do). Additionally, learning new terminology like ‚ÄúFork‚Äù and ‚ÄúCheckout‚Äù.

All of that changed today and if you have been fillind similar fears will change for you too.

All you need to do is to head out the first-contribution repository on GitHub and follow the direction. This will be sufficient enough. For the sake of learning the material further and additionally learning to explain myself clearly I will write a short instruction here. You are welcome to follow it and if you have any comments please let me know at on of the Social Networks under the site title. Thanks, enjoy!

  1. Go to the first-contribution repository on GitHub.

  2. Fork the Github by pressing the ‚ÄúFork‚Äù button on the top right corner. This will copy over the repository to your account. That way you can make any changes you want. Apologies for the lack of images. I have yet to resolve the issue with image uploads on Markdown editor for Django.

  3. Head over to the repository on you account and press the Clone or Download button (It is loacted in the top rights and is filled in with Green). You will be provided with the link to clone the repository to your computer. Copy that link.

  4. Go to the ‚ÄúTerminal‚Äù (on Mac) or the Promt Shell (on Windows, search for ‚Äúcmd‚Äù and press enter), navigate to the folder you want to keep your GitHub repositories, by using cd command. I suggest making a GitHub folder by runnning mkdir GitHub command on the Mac machine and md GitHub on Windows.

  5. Run git clone <link you copied earlier>. If you do not yet have git installed on your machine, head over to the Git Install tutorial page and then to the GitHub setup page. This will download the repository to your computer. There is one more thing you need to do before you can change the contents.

  6. Navigate to the downloaded folder with cd first-contributions and create a new branch with git checkout -b <add-your-new-branch-name> Branch will spearate your corrections fomr the original repository, which you can later submit. The owner of the original repository will decide if your additions should be ‚Äúpulled‚Äù.

  7. Add your name to the ‚ÄúContributors.MD‚Äù in text-editor of your choosing. Atom for me.

  8. Commit the changes you made with the following commands:

git add contributors.md
git commit -m "{write your message}"
git push origin <add-your-new-branch-name>
